# UI Design Rules

These rules apply ONLY when generating UI or frontend code.

If the task is backend related, ignore these rules.

---

# Visual Style

The UI should feel like modern SaaS platforms such as:

- Stripe
- Linear
- Vercel
- Notion

Style must be:

- clean
- professional
- minimal
- high trust
- enterprise friendly

Avoid:

- childish UI
- excessive animations
- clutter
- default browser styling

# Accessibility

UI must follow accessibility standards.

Rules:

- strong color contrast
- readable font sizes
- visible focus states
- large enough buttons
- do not rely on color alone for status

---

# Layout

Pages should use:

- max width containers
- good whitespace
- grid layouts
- card based design

Background

#F4F8FC

Cards

white with light border and soft shadow.

---

# Buttons

Primary button

background #1F6FB5  
text white

Secondary button

white background  
border #D9E2EC

Buttons must have hover states.

---

# Inputs

Inputs must include:

- labels
- focus state
- validation feedback

Avoid browser default styling.

---

# Tables

Tables should include:

- clear header
- status badges
- action column

Spacing must be readable.

---

# UI Behavior

Use subtle interactions:

- hover
- focus
- loading states

Avoid flashy animation.
